SuperSEO

Description

The React SEO component that handles metadata so you don't have to. Supports Google, OpenGraph, Twitter, and beyond. ðŸš€

Supports the following:

  • title and description
  • OpenGraph and Twitter Cards
  • ISO 639-1 lang attribute in html element
  • Google site verification
  • Robots and googlebot configuration
  • Google-specific meta tags
  • Charset, viewport, and contentType
  • Rating (for adult content)
  • Everything else! Just wrap custom meta tags with SuperSEO

Uses sensible fallbacks where possible and features automatic MIME type inference for media URLs.

Includes full TypeScript type definitions and rich documentation, inside and out. ðŸ™Œ

Usage

import { SuperSEO } from "react-super-seo";

Provide as little as a title and description, and go from there.

The SuperSEO component is being used on this page. Try changing the title below.
<div>
  The SuperSEO component is being used on this page. Try changing the title
  below.
</div>
<SuperSEO
  title="SEO | React Super SEO"
  description="React SEO component with OpenGraph and Twitter Cards support."
  lang="en"
  openGraph={{
    ogImage: {
      ogImage: 'http://placekitten.com/1200/630',
      ogImageAlt: 'Kittens',
      ogImageWidth: 1200,
      ogImageHeight: 630,
      ogImageType: 'image/jpeg',
    },
  }}
  twitter={{
    twitterSummaryCard: {
      summaryCardImage: 'http://placekitten.com/1200/630',
      summaryCardImageAlt: 'Kittens',
      summaryCardSiteUsername: 'justinmahar',
    },
  }}
/>
Note: You can edit the source above. Your changes will be rendered immediately.

openGraph Prop (OpenGraphMetadata)

The openGraph prop is an object with any of the following properties:

PropertyTypeDescription
ogTitlestring(Required) og:title - The title of your object as it should appear within the graph, e.g., "The Rock".
ogTypestring(Required) og:type - The type of your object. Supports article, book, profile, website. Specify the related properties in ogTypeObject. If another type is provided, use a MetaPropertiesFunction for the ogTypeObject to specify your meta properties for the type. This function should return an array of MetaProperty in the format {property: string, content: string}.
ogTypeObjectOpenGraphTypeArticle | OpenGraphTypeBook | OpenGraphTypeProfile | OpenGraphTypeWebsite | MetaPropertiesFunction(Required) The meta properties object for the type specified. Use {} to default to "website".
ogImageOpenGraphImage | OpenGraphImage[](Required) og:image - An image URL which should represent your object within the graph. Provide an array for multiple images.
ogUrlstring(Required) og:url - The canonical URL of your object that will be used as its permanent ID in the graph, e.g., "http://www.imdb.com/title/tt0117500/".
ogAudioOpenGraphAudio | OpenGraphAudio[](Optional) og:audio - Audio file(s) to accompany this object.
ogDescriptionstring(Optional) og:description - A one to two sentence description of your object.
ogDeterminer'a' | 'an' | 'the' | '' | 'auto'(Optional) og:determiner - The word that appears before this object's title in a sentence. An enum of (a, an, the, "", auto). If auto is chosen, the consumer of your data should chose between "a" or "an". Default is "" (blank).
ogLocalestring(Optional) og:locale - The locale these tags are marked up in. Of the format language_TERRITORY. Default is en_US.
ogLocaleAlternatestring[](Optional) og:locale:alternate - An array of other locales this page is available in.
ogSiteNamestring(Optional) og:site_name - If your object is part of a larger web site, the name which should be displayed for the overall site. e.g., "IMDb".
ogVideoOpenGraphVideo | OpenGraphVideo[](Optional) og:video - Video file(s) that complement this object.
fbAppIdstring(Optional) fb:app_id - The Facebook App ID lets Facebook know the identify of your site, which provides additional benefits like social analytics, comments moderation and authentication capabilities to your site.

google Prop (GoogleMetadata)

These are special tags that Google understands. They help control how your site's pages appear in Search.

Provide an object containing any of the following properties:

PropertyTypeDescription
noSiteLinksSearchBoxboolean(Optional) When users search for your site, Google Search results sometimes display a search box specific to your site, along with other direct links to your site. This tag tells Google not to show the sitelinks search box.
noTranslateboolean(Optional) When Google recognizes that the contents of a page are not in the language that the user is likely to want to read, Google often provides a link to a translation in the search results. In general, this gives you the chance to provide your unique and compelling content to a much larger group of users. However, there may be situations where this is not desired. This meta tag tells Google that you don't want us to provide a translation for this page.
noPageReadAloudboolean(Optional) Prevents web browsers from reading aloud the tagged page using the Google Assistant voice commands "Read this page" and "Read it."

Custom Meta Tags

There are many more meta tags not covered here, such as the once popular, now obsolete keywords meta tag (now ignored by search engines), and others which you may be interested in but are outside the scope of this project.

If you need something not covered above, you can always provide them as children of SuperSEO.

Anything you wrap in the SuperSEO component will be rendered to the head element for you. Sky's the limit. ðŸš€

<SuperSEO
  title="My Page"
  description="An awesome page that does a lot of cool stuff"
>
  <meta name="custom-meta" content="my-value" />
  <meta property="custom-meta-2" content="my-other-value" />
</SuperSEO>

Add your custom meta tags inside the SuperSEO component.

Examples

Fallbacks

The following shows how fallbacks are used for OpenGraph:

<SuperSEO
  title="SEO | React Super SEO"
  description="React SEO component with OpenGraph and Twitter Cards support."
  lang="en"
/>

This will be rendered as:

<html lang="en" data-react-helmet="lang">
  <head>
    <title>SEO | React Super SEO</title>
    <meta
      name="description"
      content="React SEO component with OpenGraph and Twitter Cards support."
      data-react-helmet="true"
    />
    <meta
      property="og:title"
      content="SEO | React Super SEO"
      data-react-helmet="true"
    />
    <meta property="og:type" content="website" data-react-helmet="true" />
    <meta
      property="og:description"
      content="React SEO component with OpenGraph and Twitter Cards support."
      data-react-helmet="true"
    />
  </head>
  ...
</html>

Note: Twitter uses OpenGraph og:title and og:description as a fallback, so rendering Twitter meta properties is not necessary here.

Specifying OpenGraph and Twitter

      summaryCardImage: "http://placekitten.com/1200/630",
